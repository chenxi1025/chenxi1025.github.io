3、JDK 和 JRE 和 JVM 的作用和区别

3.1、JDK(Java SE Development Kit)

  Java标准开发包。它提供了编译、运行Java程序所需的各种工具和资源，包括编译器、Java运行环境及常用的Java类库；

3.2、JRE(Java Runtime Environment)

  Java运行环境。用于解释执行Java的字节码文件。普通用户只需要安装JRE用来运行Java环境。而开发人员需要安装JDK来编译、调试程

3.3、JVM(Java Virtual Mechinal)

  Java虚拟机，是JRE的一部分。它是实现整个Java跨平台特性的最核心部分，是可运行Java字节码文件的虚拟计算机。所有平台上的JVM只需要向编译器提供相同的接口，而编译器只需要面向JVM生成能识别的代码，然后由虚拟机来解释执行。

3.4、三者的区别和联系

  1. 区别
    JDK用于开发；JRE用于运行Java程序
  2. 联系
    JDK包含JRE，JDK和JRE都包含JVM

11.12、序列化和反序列化

定义：
  Java对象转换为字节序列的过程称为对象的序列化
  字节序列恢复成Java对象的过程称为对象的反序列化
作用：
  1、持久化：把对象永久的保存在硬盘上，通常放在一个文件中，比如：休眠的实现
  2、网络通信：在网络上传送对象的字节序列。比如：服务器之间的数据通信、对象传递
ex：将Person类的实例进行序列化和反序列化

对象流入坑指南：
1、对象流不同于普通的字节流，当对象流中没有数据时，程序如果尝试读取数据，汇报EOFException;而普通字节流就不会出现这样的问题，字节流会返回-1
2、最好在实际使用的过程中，我们先实例化ObjectOutputStream，再实例化ObjectInputStream(如本例中的实例化顺序和之前普通字节流实例化顺序的不同)，这是由这两个类的设计思想所决定的。如此能保证同一资源的对象流ObjectInputStream能够及时读取到序列化头而不至于阻塞或者引发EOF异常
3、static属性不参与序列化
4、对象中某些属性如果不想被序列化，使用transient修饰即可
5、为了防止读和写的序列化ID不一致，一般指定一个固定的序列化ID

12.4、静态内部类

12.4.1、概念

使用static来修饰一个内部类，则这个内部类就属于外部类本身，而不属于外部类的某个对象。称为静态内部类。

12.4.2、特点

1、非静态内部类中不允许定义静态成员
2、静态内部类的创建是不需要依赖于外部类，可以直接创建
3、静态内部类不能访问外部类的实例成员，只能访问外部类的类成员
4、一个类如果只用一次的话可以将其改写成内部列，提升性能
5、外部类不使用的话内部类不会被加载(不会编译)

12.4.3、静态内部类的加载时机

1、外部类初次加载，会初始化静态变量、静态代码块、静态方法，但不会加载内部类和静态内部类。
2、直接调用静态内部类时，外部类不会被加载

12.6、线程状态


注意：
1、线程进入“阻塞状态”后不会马上回到“运行状态”，会重新进入“就绪状态”；
2、线程进入“死亡状态”后不能重新开启。如果重新开启那么线程就是一个“新的线程”，不再是“原先的线程”了

12.7、线程状态详解

  • NEW：线程刚被创建(新生状态)
  • RUNNABLE：线程处于可执行状态(就绪状态) 线程进入就绪状态的几种情况
    1、调用start()方法;
    2、阻塞事件解除，线程重新进入就绪状态;
    3、调用yield()方法让出cpu的调度权，直接重新进入就绪状态;
    4、jvm将本地线程切换到其他线程时，本地线程也会进入就绪状态(了解)
  • Running：线程处于运行状态
  • BLOCKED、WAITING：线程被阻塞，阻塞状态
    线程进入阻塞状态的几种情况
    1、调用sleep(long millSeconds)方法
    2、调用wait()方法：相当于“红绿灯”，让别的行人(车)先过。让出cpu的调度权
    3、调用join()方法：相当于插队;
    4、有些特殊的操作会进入阻塞状态，如：IO流操作。read,write需要操作系统操作，其所在线程也会进入阻塞状态
  • TERMINATED：线程执行结束，被终止，死亡状态
    线程进入死亡状态的几种情况
    1、线程代码执行结束
    2、线程被强制终止：stop()和destory()方法，但是并不推荐使用
    3、人工干涉

15、网络编程

15.1、网络编程入门

15.1.1、软件结构

  • C/S结构：全称为Cilent/Server结构，是指客户端和服务器的结构。常见程序有QQ、迅雷等软件

  • B/S结构：全称为Browser/Server结构，是指浏览器端和服务器的结构。常见浏览器为Chrome、Firefox和IE等

两种架构各有优势，但是无论哪种架构，都离不开网络的支持。网络编程，就是在一定的协议下，实现两台计算机的通信的程序

15.1.2、网络通信协议

  • 网络通信协议：通信协议是对计算机必须遵守的规则，只有遵守这些规则，计算机之间才能进行通信。这就好比在道路中行驶的汽车一定要遵守交通规则一样，协议中对数据的传输格式、传输速率、传输步骤等做了统一规定，通信双方必须同时遵守，最终完成数据交换。

  • TCP/IP协议：传输控制协议/因特网互联协议(Transmission Control Protocol/Internet Protocol)，是Internet最基本、最广泛的协议。它定义了计算机如何连入因特网，以及数据如何在它们之间传输的标准。它的内部包含一系列的用于处理数据通信的协议，并采用了4层的分层模型，每一层都呼叫它的下一层所提供的协议来完成自己的需求。

上图中，TCP/IP协议中的四层分别是应用层、传输层、网络层和链路层，每层分别负责不同的通信功能。
链路层：链路层是用于定义物理传输通道，通常是对某些网络连接设备的驱动协议，例如针对光纤、网线提供的驱动。
网络层：网络层是整个TCP/IP协议的核心，它主要用于将传输的数据进行分组，将分组数据发送到目标计算机或者网络。
运输层：主要使网络程序进行通信，在进行网络通信时，可以采用TCP协议，也可以采用UDP协议。
应用层：主要负责应用程序的协议，例如HTTP协议、FTP协议等。

15.1.3、协议分类

通信的协议还是比较复杂的，java.net 包中包含的类和接口，它们提供低层次的通信细节。我们可以直接使用这些类和接口，来专注于网络程序开发，而不用考虑通信的细节。

java.net 包中提供了两种常见的网络协议的支持：

  • UDP：用户数据报协议(User Datagram Protocol)。UDP是无连接通信协议，即在数据传输时，数据的发送端和接收端不建立逻辑连接。简单来说，当一台计算机向另外一台计算机发送数据时，发送端不会确认接收端是否存在，就会发出数据，同样接收端在收到数据时，也不会向发送端反馈是否收到数据。

    由于使用UDP协议消耗资源小，通信效率高，所以通常都会用于音频、视频和普通数据的传输例如视频会议都使用UDP协议，因为这种情况即使偶尔丢失一两个数据包，也不会对接收结果产生太大影响。

    但是在使用UDP协议传送数据时，由于UDP的面向无连接性，不能保证数据的完整性，因此在传输重要数据时不建议使用UDP协议。UDP的交换过程如下图所示。

    特点:数据被限制在64kb以内，超出这个范围就不能发送了。

    数据报(Datagram):网络传输的基本单位

  • TCP：传输控制协议(Transmission Control Protocol)。TCP协议是面向连接的通信协议，即传输数据之前，在发送端和接收端建立逻辑连接，然后再传输数据，它提供了两台计算机之间可靠无差错的数据传输。

    在TCP连接中必须要明确客户端与服务器端，由客户端向服务端发出连接请求，每次连接的创建都需要经过“三次握手”。

    • 三次握手：TCP协议中，在发送数据的准备阶段，客户端与服务器之间的三次交互，以保证连接的可靠。

      • 第一次握手，客户端向服务器端发出连接请求，等待服务器确认。
      • 第二次握手，服务器端向客户端回送一个响应，通知客户端收到了连接请求。
      • 第三次握手，客户端再次向服务器端发送确认信息，确认连接。整个交互过程如下图所示。

      完成三次握手，连接建立后，客户端和服务器就可以开始进行数据传输了。由于这种面向连接的特性，TCP协议可以保证传输数据的安全，所以应用十分广泛，例如下载文件、浏览网页等。

15.1.4、网络编程的三要素

协议

  • 协议：计算机网络通信必须遵守的规则

IP地址

  • IP地址：指互联网协议地址（Internet Protocol Address）。俗称IP。IP地址用来给一个网络中的计算机设备做唯一的编号。假如我们吧“个人电脑”比作“一台电话”的话，那么“IP地址”就相当于电话号码

  • IP地址的分类

    • IPv4：是一个32位的二进制数(1个字节8位)，被分为4个字节，标识成a.b.c.d的形式，例如192.168.65.100。其中a、b、c、d都是0~255之间的十进制整数，那么最多可以表示2^32≈42e个

    • IPv6：由于互联网的蓬勃发展，IP地址的需求量愈来愈大，但是网络地址资源有限，使得IP的分配越发紧张。
      为了扩大地址空间，拟通过IPv6重新定义地址空间，采用128位地址长度，每16个字节一组，分成8组十六进制数，表示成ABCD:EF01:2345:6789:ABCD:EF01:2345:6789，号称可以为全世界的每一粒沙子编上一个网址，这样就解决了网络地址资源数量不够的问题。

15.2、TCP通信程序

15.2.1、概述

TCP通信能实现两台计算机之间的数据交互，通信的两端，要严格区分客户端(Client)和服务端(Server)。

两端通信时步骤：
1、服务端程序，需要事先启动，等待客户端的连接；
2、客户端主动连接服务器端，连接成功才能通信。服务端不可以主动连接客户端

在Java中，提供了两个类用于事先TCP通信程序：

  • 客户端：java.net.Socket 类表示。创建Socket对象，向服务端发出连接请求，服务端响应请求，两者建立连接开始通信。
  • 服务端：java.net.ServerSocket 类表示。创建ServerSocket对象，相当于开启一个服务，并等待客户端的连接。

TCP通信分析图解

  1. 【服务端】启动,创建ServerSocket对象，等待连接。
  2. 【客户端】启动,创建Socket对象，请求连接。
  3. 【服务端】接收连接,调用accept方法，并返回一个Socket对象。
  4. 【客户端】Socket对象，获取OutputStream，向服务端写出数据。
  5. 【服务端】Scoket对象，获取InputStream，读取客户端发送的数据。

到此，客户端向服务端发送数据成功。

自此，服务端向客户端回写数据。

  1. 【服务端】Socket对象，获取OutputStream，向客户端回写数据。
  2. 【客户端】Scoket对象，获取InputStream，解析回写数据。
  3. 【客户端】释放资源，断开连接。
